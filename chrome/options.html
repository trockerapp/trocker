Extension Options
Block Trackers:

Clarification about permissions: the following key features are why Trocker needs the <all_urls> permission to function:

  • Blocking email trackers in any webmail that does not proxy images on its own server (e.g. outlook.com)
  • Bypassing tracked links (links that track when you click on them)

Without the <all_urls> permission, Trocker will only be able to block tracking images (but not links) and only in some webmails (e.g. Gmail) because the browser will not let Trocker know about all tracking images/links and thus Trocker will not be able to block them. This is a powerful permission which is why browsers rightfully warn you that the extension can "Read and change all your data on the websites you visit".

Show Current Tab Trackers Count:

Shows the number of tracking links found in the current tab on the Trocker icon.
(The number includes invisible trackers loaded in the background and also tracked links)

Expose Trackers:

  • Hidden Tracking images will become visible as (block-listed trackers) or (auto-detected trackers)
  • Mouse pointer turns into over tracked links.
Any website:

This feature requires the following permissions to be granted:

Will try to count and expose trackers on any website that attempts tracking (e.g. will start running on mail.example.com after the first time a tracked image is loaded from it). Even without this option, all block-listed trackers will still be blocked.

Verbose Log:

Extensively logs activities in console.

Customize lists:

Use custom lists for known trackers.

If anything breaks, simply disable "Customize lists" or keep the enabled but press "Restore default" and "Save" on both text boxes.
